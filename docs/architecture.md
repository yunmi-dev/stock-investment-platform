# 시스템 아키텍처 초안

## 전체 구조

```
[Flutter App/Web]
        ↓
    [Nginx]
        ↓
  [Spring Boot]
    ↓     ↓
[Redis] [PostgreSQL]
```

## 기술 선택 이유

### PostgreSQL
- 트랜잭션 안정성 (금융 데이터)
- 복잡한 쿼리 지원

### Redis
- 주가 캐싱
- 주문 시 동시성 제어용으로 분산 락 구현

### Spring Boot
- 빠른 개발
- 풍부한 생태계

## 캐싱 전략 (예정)
- Redis: 주식 시세 (TTL 10초)
- 로컬 캐시 추가 검토 중
